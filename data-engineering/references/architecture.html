Data Engineering Skill

What it is: An interactive pipeline builder that creates ETL pipelines through conversation. You provide data, it asks questions, builds the pipeline, validates quality, loads into your database, and optionally pushes to GitHub.

What it replaces: Writing boilerplate extraction, transformation, quality checks, and loading code from scratch every time you have a new dataset.

Why This Skill Exists

Building a data pipeline typically requires:

  • Reading the source file and understanding its schema
  • Cleaning column names, parsing dates, removing duplicates
  • Checking for nulls, invalid formats, and data freshness
  • Connecting to a database and loading data
  • Saving the code so you can run it again
  • Pushing to Git for version control

This skill does all of that through a guided conversation. You answer questions, it does the work.

The skill follows 8 stages. At each stage, it asks you questions before proceeding. Nothing happens without your approval.

Re-running the pipeline

The saved script supports command-line arguments:

python3 etl_kc_house_data_to_postgres.py \
  --dataset /path/to/new_data.csv \
  --table kc_house_data \
  --database kaggle_data \
  --user postgres \
  --password password

Run python3 etl_kc_house_data_to_postgres.py --help to see all options.
